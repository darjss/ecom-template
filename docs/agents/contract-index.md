# Contract authority index

Use this index before changing a feature. It points to contract owners; it does not restate their bodies.

## Precedence

1. The current [ecommerce system specification](../specs/ecommerce-system.md), mirrored in [issue #33](https://github.com/darjss/ecom-template/issues/33), wins wherever it records a later scope or simplification decision.
2. The [final scope reconciliation](../wayfinder/final-scope-reconciliation.md) wins over older artifacts wherever the current specification does not replace it.
3. The accepted feature contract governs the remaining details for its feature.
4. Research, prototypes, the [landed bootstrap plan](../architecture/bootstrap-plan.md), and other historical artifacts provide evidence or implementation navigation only. They do not override an accepted contract.

Issue #33 and the repository specification must be updated together. If two accepted contracts still conflict after applying this precedence, stop and report the exact clauses rather than choosing silently.
