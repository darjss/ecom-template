# UI review — Өрнүүн 48 storefront + Admin (HEAD `90f506e`)

**Date:** 2026-07-17
**Method:** code review + real browser proof (agent-browser) against the running app through Portless — production preview (`pnpm preview`) at `https://urnuun-48.shop.localhost`, desktop 1280px and mobile 390px. Contrast ratios computed with a validated OKLCH→luminance converter (checked against white/black/#ff0000 references). One synthetic published product was inserted into the **local disposable** D1 to render the product page and catalog list. Admin dashboard itself is Google-OAuth gated and was not entered; the login/awaiting/gate surfaces were.
**Authorities:** PRODUCT.md, DESIGN.md (Өрнүүн 48 Pantry Shelf), `impeccable` audit dimensions, WCAG 2.2 AA, spec stories 1–15 and 25–28.

## Verdict

The **art direction is genuinely good** — the pantry-shelf composition (beams, plinth, bays, shelf-marker badge, cohesive generated imagery) is distinctive, merchant-authentic, and not template sameness. It honors PRODUCT.md's "adapt composition, not just decoration."

The **execution is not finished**. The dev server cannot render images or any interactive island (the repo's own proof path is broken), the purchase control is a permanently disabled placeholder, the featured-catalog section is a navy void with a squished text link, the signature tomato color fails contrast for body text, and the implementation has drifted from its own DESIGN.md tokens.

## P0 — blocking

### [P0] Dev server 404s every static asset and every island module
**Location:** `apps/urnuun-48/wrangler.jsonc` (`assets: { run_worker_first: true }` with no `directory`), `astro.config.mjs` (stray `fetchFile: null` no-op key).
**Evidence:** through Portless dev, `/media/hero-pantry.webp` → 404, `/favicon.svg` → 404, `/@id/@ecom/admin` → 404, `/@id/@astrojs/solid-js/client.js` → 404. The login page renders **without its Google sign-in button**; the homepage renders a giant empty hero panel. Production preview (`pnpm preview`) serves everything correctly.
**Impact:** the repo's Proof gate ("run the app through Portless, inspect affected browser behavior") cannot prove any UI with images or JavaScript. PRs #73–#75 could not have been browser-proved in dev — and it shows (see the P1s below).
**Recommendation:** fix the dev assets pipeline (give `assets` its directory or make the adapter serve `public/` in dev) and delete the meaningless `fetchFile: null`. This is the first thing to repair; every other UI proof depends on it.

### [P0] The purchase control is a dead placeholder — and the Cart island is unused
**Location:** `apps/urnuun-48/src/pages/products/[slug].astro` — `<button type="button" disabled>Боломж шалгаж байна</button>`; `packages/storefront/src/CartIsland.tsx` is defined but imported by no page.
**Impact:** "Checking availability…" claims something is happening; nothing ever checks. A shopper can never buy. Spec stories 26–27 require the purchase island to start disabled, fetch live availability, and fail closed — the shell shipped without the island.
**Recommendation:** wire the availability/Cart island (the accepted spec seam), and until then render an honest static label ("Захиалга удахгүй") instead of a fake pending state. Delete or wire `CartIsland` — dead UI code violates the deletion test.

## P1 — major

### [P1] Product name renders unstyled
**Location:** `apps/urnuun-48/src/styles/global.css` — the display rule targets `.product-copy h2, .story-page h1`, but `[slug].astro` uses `<h1 id="product-title">` inside `.product-copy`.
**Impact:** the most important text on the product page gets browser-default sizing (verified in screenshot: "Төгс тоть" smaller than the price below it).
**Fix:** change the selector to `.product-copy h1, .product-copy h2, .story-page h1` (or fix the markup).

### [P1] Featured catalog section is a 42rem void with a squished link
**Location:** `apps/urnuun-48/src/pages/index.astro` + `.featured`/`.product-copy` in `global.css`.
**Impact:** `min-height: 42rem` of navy with one tiny bullet at the top-right; product name and price run together ("Төгс тоть15,900 ₮") because `<strong>` and `<span>` are bare inline elements; no imagery, no price hierarchy, no affordance. With 9 products it's still a plain bulleted list in a void. This is the section that sells — currently the weakest thing on the page. The `products.length === 0` empty state is bare text.
**Fix:** a real catalog list: each row with name, integer-MNT price on its own tabular baseline, and generous hit area; drop the fixed 42rem; give the empty state designed copy.

### [P1] Three contrast failures on the signature palette (WCAG AA)
**Measured (validated converter):**

| Pair | Ratio | Needs | Where |
| --- | --- | --- | --- |
| Cream body copy on tomato hero | **3.85:1** | 4.5:1 | `.pantry-copy > p` |
| Tomato kicker/labels on paper | **4.22:1** | 4:5→4.5:1 | `.kicker`, `.pantry-categories p/span` (0.68rem text) |
| Tomato label on yellow bundle bay | **3.20:1** | 4.5:1 | `.bundle-bay p` |

**Fix:** darken toward `tomato-deep oklch(45% 0.17 31)` (already in DESIGN.md, unused in CSS) for small text on paper/yellow; raise hero body copy to paper-white or deepen the tomato block. Large display text (h1 paper-on-tomato 4.22:1, yellow-on-navy 10.1:1, product-copy-on-navy 9.5:1) passes and needs no change.

### [P1] Hardcoded categories that go nowhere
**Location:** `index.astro` — `const categories = [...]` renders five `<li>` with no link.
**Impact:** a shelf of fake doors; shoppers will tap "Хүнс" and get nothing. Either they navigate (make them anchors) or they're decor (remove the list semantics and visual affordance).

### [P1] Hardcoded product image + empty alt
**Location:** `[slug].astro` — every product shows `/media/p07-tote-sand.webp` with `alt=""`.
**Impact:** wrong product imagery once a second product exists; the product's primary image is invisible to screen readers.
**Fix:** per-product media is already the accepted contract direction (media/CMS seam); until then, at minimum give the image meaningful alt text from the product name.

### [P1] DESIGN.md token drift
**Location:** `packages/ui/src/styles.css` + `apps/urnuun-48/src/styles/global.css` vs DESIGN.md.
- Accent: CSS `oklch(58% 0.16 38)` vs token `tomato oklch(55% 0.19 33)`; navy `oklch(28% 0.045 252)` vs `trust-navy oklch(31% 0.07 249)`; ink hue 66 vs 48.
- Radius: token `control: 8px`; buttons ship as `999px` pills; `--radius: 1.15rem` matches nothing in the scale.
- Letter-spacing: token display `-0.035em`; CSS ships `-0.08em` (h1) and `-0.06em` (h2) — visibly cramped, below the -0.04em readability floor.
- Font: token `Arial, Noto Sans, system-ui`; CSS asks for `Inter` but never loads it (no @font-face, no fontsource) → silently falls back.
- Two overlapping token sources (`ui/styles.css :root` and app `global.css`) define neighboring-but-different palettes.

**Fix:** make DESIGN.md and the CSS agree — either import the tokens verbatim or amend DESIGN.md. One source of truth.

## P2 — minor

- **[P2] Orphaned ₮:** `120,000 ₮` wraps after the number at desktop widths (`.delivery-bay span` clamp is too big for the bay). Use `white-space: nowrap` or `&nbsp;`.
- **[P2] Inconsistent wordmark:** home/products render the `48` badge + "Өрнүүн"; story renders plain text — yet both carry `transition:name="wordmark"`, so the View Transition morphs between different marks.
- **[P2] `client:only` islands with no shell:** without JS (or with dev's 404s), the login page has no button and the customer-auth section is a blank panel. Render at least a static noscript note ("Энэ үйлдэлд JavaScript шаардлагатай").
- **[P2] Touch targets:** header nav links are bare text (~20px tall) — under the 44px guidance on mobile.
- **[P2] Kicker-per-section:** the tiny-uppercase-tracked eyebrow sits above every section. The shelf-label motif partially excuses it, but the cadence is AI-grammar; vary it (the story page and featured section could stand without).
- **[P2] No responsive images:** one 1000px WebP serves every viewport; fine at 54–94KB today, but the R2 resizing seam should own `srcset` when media lands.

## P3 — polish

- Featured empty state ("Одоогоор нийтлэгдсэн бүтээгдэхүүн алга.") deserves designed copy, not a bare sentence in a navy void.
- The global reduced-motion `!important` kill-switch also murders harmless opacity fades; prefer reducing to crossfade.
- `.featured` CSS is written for `.product-visual + .product-copy` (product page) but the homepage reuses the class with two `.product-copy` children — the class name now lies about its contents.
- The disabled purchase button has no `aria-describedby` explaining why it's disabled.

## Positive findings — keep and replicate

- **The shelf composition works.** Beam/plinth woodwork, numbered "Тасалгаа" rails, the `48` shelf-marker badge, the multiply-blend bundle bay — this is a real art direction, not a reskin.
- **Generated imagery is cohesive** — same warm pantry light across hero, bundle, and product shots; sized WebP (42–94KB) with dimensions, lazy loading below the fold, `fetchpriority` on the hero.
- **Accessibility scaffolding is real:** skip link (verified working with 3px focus-blue outline on the second Tab), `:focus-visible` token, `aria-current`, `aria-live` on cart and auth feedback, `lang="mn"`, `inputmode`/`autocomplete="one-time-code"` on the OTP form, semantic landmarks.
- **Commerce behavior is honest where it exists:** OTP flow degrades gracefully when the SMS gateway is unavailable (verified: polite Mongolian failure message, and the rate limiter really returns 429 + retry-after), the demo notice is unmissable, prices are integer MNT with proper `mn-MN` grouping.
- **Mobile reflow is clean:** zero horizontal overflow at 390px, categories become a scroll strip, beams simplify, the auth panel stacks.
- **The header/gate discipline:** admin → 303 → login, `private, no-store` everywhere non-public, host validation.
